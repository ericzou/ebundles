Subversion Help

This document describes the commands of the TextMate Subversion bundle and how you can fine-tune their behavior. For general Subversion help and tutorial, you should read the Subversion book.

Commands

Add to Repository
selected project files/folders or the active file if it doesn't belong to a project
No
Schedules the targets for addition to the repository.
Blame
active file
Yes
Displays a line-by-line history of the file, showing you who wrote which line in what revision. Click a line to jump to it in an editor window. Hover over a revision number or author name to see when the corresponding line was last changed. The date format is adjustable.
Commit
selected project files/folders or the active file if it doesn't belong to a project
Yes
Commits your changed files to the repository. A dialog asks you for the description of your changes; you may also choose to exclude files from the commit by unchecking them. If no files are selected or active, this command does nothing. If the target files have no local changes, nothing happens.
Diff Revisions…
active file
Yes
Displays the differences between two specific revisions of the active file. You will be presented with a list of revisions; please select two.
Diff With Newest (HEAD)
active file
Yes
Displays the differences between the active file and the newest available revision of the file on the server. Equivalent to svn diff -rHEAD.
Diff With Working Copy (BASE)
active file
No
Displays the differences between the active file and an unaltered, pristine copy of the file at the same revision. Equivalent to svn diff -rBASE.
Diff With Previous Revision (PREV)
active file
Yes
Displays the differences between the active file and the previous revision of the file. Equivalent to svn diff -rPREV.
Diff With Revision…
active file
Yes
Displays the differences between the active file and a different revision of the same file. This command presents you with a list of revisions to choose from.
Info
selected project files/folders or the active file if it doesn't belong to a project
No
Displays detailed information about the selected files, including the type of file, who last changed the file, the date file was last changed, the repository URL to the file, and other information. Some parameters are configurable via shell variables; see the Configuration Options section below.
Log
selected project files/folders or the active file if it doesn't belong to a project
Yes
Displays the commit message history for the selected files. Some parameters are configurable via shell variables; see the Configuration Options section below.
Resolved
active file
No
Removing the conflict-related artifact files, allowing the file to be committed. Does not remove conflict markers or resolve textual conflicts.
Revert
active file
No
Reverts the file to the base revision. Any modifications to the file will be lost.
Show All Modifications
selected directory in project drawer, otherwise directory of active file
No
Displays the differences for all locally modified files, compared with their unaltered, pristine copy.
Status
directory of active file
No
Displays a list of files with changes in your working copy, along with the type of change for each file.
Update to Newest
selected project files/folders or the active file if it doesn't belong to a project
Yes
Updates the targets with the newest changes from the repository.
Update Entire Project to Newest
project
Yes
If there is a project open, updates all of the files and folders in the project with the newest changes from the repository. (The folders must be rooted beneath TM_PROJECT_PATH. Projects containing only groups created in TextMate won't work.)
View Revision…
active file
Yes
Displays a different revision of the active file. This command presents you with a list of revisions to choose from.

Configuration Options

These shell variables allow you to tweak the behavior of the certain commands if need be. The default values should make sense for normal use.

$TM_SVN
svn
the path to your svn executable.
$TM_SVN_DIFF_CMD
not set
Allows you to set a command that should be used to present file differences. If you want to have differences shown using Appleâ€™s FileMerge then you can use fmdiff.
$TM_RUBY
ruby
Here you can tweak with what Ruby the formatting scripts will be executed.
$TM_SVN_DATE_FORMAT
not set (No Changes, just show what SVN shows)
A date format string that allows you to tweak the format in which Blame, Log and Info show you dates. See strftime(3) for what you can enter here.
$TM_SVN_CLOSE
false
With this option you can adjust whether the windows of Blame and Info close if you click on a link which opens a file in TM. Set it to 1 or true if windows should close or to something else if they should not.
$TM_SVN_LOG_RANGE
BASE:1
The default range to query for log messages.
$TM_SVN_LOG_LIMIT
15
The number of messages to show. 0 means no limit.
$TM_SVN_INFO_HIDE
not set (nothing)
Here you can adjust what keys of the info stream you don't want to see, it is case insensitive and a comma (,) separated list. If you enter a *, it is assumed that you want to hide all vars, if so, you can make some values visible again with $TM_SVN_INFO_SHOW.
$TM_SVN_INFO_SHOW
not set (everything else)
This is also a comma separated list of keys, it just makes sense if you did hide all with the above var. * has no special meaning here. Everything you enter here will be shown if you want to hide everything else.
$TM_SVN_IGNORE_BAD_LINES
not set
By setting this variable, Log will not print bad (unexpected) lines from the beginning of subversions output, this could be usefull if you always get this output but its nothing critical for your situation.
